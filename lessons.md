# Vanguard Agent Lessons

Pamięć agenta o napotkanych problemach i ich rozwiązaniach między sesjami. Przed rozpoczęciem każdego zadania sprawdź te wpisy, aby nie powtarzać błędów.

| Data | Zadanie | Problem | Lekcja |
|---|---|---|---|
| 2026-06-14 | Weekly Board implementation | Arithmetic subtraction of Date objects directly causes type errors in TypeScript (TS2362 / TS2363). | Zawsze używaj `.getTime()` przy odejmowaniu lub sortowaniu obiektów `Date` w TS. |
| 2026-06-14 | Weekly Board implementation | Type inference of inline declared patch object blocks dynamic assignment of optional properties (like `due_date` when defined initially without it). | Zadeklaruj jawny typ obiektu (np. `patch: { ai_bucket: string; due_date?: string | null }`) przy tworzeniu patcha. |
| 2026-06-14 | UI component compile validation | `SectionTitle` component defined parameters mismatching inline props invocation without `action`. | Zawsze w definicji propsów komponentu w TS zaznaczaj parametry jako opcjonalne (`action?: any`) lub dodawaj wartości domyślne. |
| 2026-06-16 | Dodanie kolumny `life_goal` do `dreams` przez MCP migration | `database.types.ts` NIE jest automatycznie regenerowany po migracji — TS rzuca błąd `not assignable to type 'never'` na nowych polach. | Castuj payload insertu/updatu jako `as any` do czasu ręcznej regeneracji typów (`supabase gen types`). |
| 2026-06-16 | Picker celów życiowych w DesktopDashboard | `as const` na tablicy tupli wewnątrz `.map()` powoduje TS2322 bo destrukturyzacja nie może przypisać do literal type. | Zamiast `as const` używaj jawnego typu `[string, string, string][]`. |
| 2026-06-20 | CheckpointsCard w Dziś | Checkpointy miały `status = 'open'` w DB ale karta filtrowała `.eq('status', 'pending')` — karta zawsze była pusta. | Sprawdzaj rzeczywiste wartości enum w DB zanim piszesz filtr statusu; użyj `.in('status', ['pending', 'open'])` jako bezpieczna forma. |
| 2026-06-20 | `/goal` pełny przegląd kodu/jakości | `MiddayCheckInCard.save()` robił `await upsert(...)` w `try` i zawsze ustawiał `setSaved(true)`, bo Supabase JS nie throwuje na błędzie Postgres/RLS — zwraca `{error}` w resolved value. Catch nigdy się nie odpalał, użytkownik widział "zapisano" nawet gdy nic się nie zapisało. | Po `supabase....().upsert/insert/update()` zawsze sprawdzaj `error` z wyniku (użyj `unwrap()` z `lib/supabaseUtils.ts`, nie goły `try/catch` — ten łapie tylko network-level throws, nie DB errors). |
| 2026-06-20 | `/goal` pełny przegląd kodu/jakości | `useHaptics()` zwracał nowy obiekt+funkcje przy każdym renderze → każdy komponent z `haptics` w deps `useEffect` dostawał `exhaustive-deps` warning i efekt re-truthował się bez sensu. | Hooki bez zależności od props/state (haptics, helpers) zwracaj jako stały obiekt zdefiniowany raz na module-level, nie twórz go w body hooka. |
| 2026-06-20 | `/goal` pełny przegląd kodu/jakości | `new Date().toLocaleDateString('en-CA', { timeZone: 'Europe/Warsaw' })` był copy-pasted ~70x w 25 plikach frontu — BUG-01 (zły timezone w `tomorrowWarsawDate`) pokazał już, że to miejsce najwyższego ryzyka regresji. | Dodano `src/lib/date.ts` (`getTodayWarsaw()`, `formatWarsawDate(d)`) i podpięto wszystkie miejsca. Nowy kod z datą Warsaw zawsze przez ten helper, nigdy inline `toLocaleDateString`. |
| 2026-06-20 | `/goal` deep bugfix pass (frontend+backend, 8 parallel review agentów) | Wiele miejsc liczyło "dziś"/zakresy dat z `new Date()` (czas lokalny browsera/serwera, nie Warsaw) i porównywało wynik z resztą apki, która jest Warsaw-anchored — `Direction.tsx` miał DWIE konkurujące funkcje `todayDate()`/`todayWarsaw()` jednocześnie używane w tym samym komponencie. `DesktopDashboard.tsx`: `body.find(b => b.date <= daysBefore(28))` na tablicy ASC zwracał najstarszy wpis (~90 dni), nie ten najbliższy 28 dniom — klasyczny błąd `.find()` na rosnącej liście przy szukaniu "najbliższego do cutoffu". | Każda funkcja licząca "dziś"/zakres dat musi przejść przez `getTodayWarsaw()`/`formatWarsawDate()` — zero wyjątków, zero równoległych lokalnych helperów. Szukając "najbliższego elementu przed/po cutoffie" w tablicy sortowanej ASC, użyj `.reverse().find()` albo iteruj od końca, nigdy `.find()` na surowej tablicy. |
| 2026-06-20 | `/goal` deep bugfix pass | Supabase `.update()/.insert()` bez `.select()` NIE da się bezpiecznie sprawdzić przez `unwrap()` z `lib/supabaseUtils.ts` — `unwrap` rzuca "No data returned" gdy `data` jest `null`, co dla plain update/insert (bez `.select()`) jest NORMALNYM wynikiem sukcesu, nie błędem. `unwrap`/`unwrapList` są tylko dla zapytań z `.select()`. | Dla zapisów bez `.select()` sprawdzaj `{ error }` ręcznie (`if (error) throw new Error(error.message)`), nie owijaj w `unwrap()`. |
| 2026-06-20 | `/goal` deep bugfix pass (Keep.tsx PGRST205 fallback) | `handleUpdate`/`handleDelete`/`handleTogglePin` w Keep.tsx ignorowały `PGRST205` (tabela nie istnieje) i mutowały tylko `setNotes()` w pamięci — bez zapisu do `localStorage('vanguard_local_keep_notes')`. Po odświeżeniu `fetchNotes()` czytał starą wersję z localStorage — edycje/usunięcia "wracały" po reload. | Każda CRUD-ścieżka z trybem fallback (localStorage/offline) musi pisać do TEGO SAMEGO storage co ścieżka odczytu (`fetchNotes`), nie tylko do React state — inaczej fallback jest tylko częściowy i myli użytkownika. |
| 2026-06-20 | Monolith refactoring (`Dashboard.tsx`, `Stats.tsx`) | Refaktoryzacja ogromnych plików poprzez wydzielanie podkomponentów (np. `NutritionCard`) pozwala uprościć strukturę. Zachowanie kontenerów wrapperów layoutu w nadrzędnym pliku pozwala uniknąć zmian stylizacyjnych i zachować pixel-perfect wygląd. | Zawsze pozostawiaj kontenery stylizujące (np. flex/grid/spacing wrappers) w głównym komponencie sterującym, a same logiczne widgety wydzielaj jako pure components zasilane przez props. |
| 2026-06-20 | ESLint empty catch blocks | Puste bloki `catch {}` przy parsowaniu JSON (np. w preferencjach profilu) powodują błędy `no-empty` w ESLint przy rygorystycznym `--max-warnings=0`. | Zawsze wstawiaj komentarz (np. `catch { /* ignore malformed pref */ }`) wewnątrz pustych bloków `catch`, aby linter traktował blok jako celowo udokumentowany, a nie pusty. |
