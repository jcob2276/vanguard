# Design System 7‚Üí10: Glass Morphism Consolidation

## Diagnosis

The iOS-inspired glass morphism direction is solid but fragmented. Four findings from exploration:

| Problem | Evidence |
|---------|----------|
| **4 of 7 glass CSS classes are dead** | `ios-glass-chrome`, `glass-panel`, `ios-modal-backdrop`, `ios-nav-bar` ‚Äî defined in `index.css`, zero TSX consumers |
| **Card variant="glass" has no blur** | `Card.tsx:24-29` ‚Äî only sets `bg-surface-1` + border + shadow. Two components (MuscleHeatmap, BrainHealth) manually add `backdrop-blur-[var(--blur-md)]` |
| **Glass hierarchy is undocumented** | DESIGN_SYSTEM.md mentions glass classes in ¬ß4 table but no decision tree for when to use which level |
| **Legacy `keep-*` classes bypass promoted `ios-*`** | `EditNoteModal.tsx` uses `keep-modal-backdrop`/`keep-ios-nav` instead of `ios-modal-backdrop`/`ios-nav-bar` |

**What's already clean (don't touch):**
- All z-index values use `--z-*` tokens (zero raw values)
- All backdrop-blur uses `--blur-*` tokens (zero raw values)
- Surface/color/text token system is solid
- Button, Badge, Tabs components are well-structured

### Step 2: Unify glass system into 3 levels with tokens

Replace the current ad-hoc glass classes with a systematic token hierarchy:

```css
/* ‚îÄ‚îÄ Glass Material Hierarchy ‚îÄ‚îÄ */
/* Level 1: Structural ‚Äî sidebar, bottom-nav (heavy, opaque) */
/* Level 2: Elevated ‚Äî header, toolbar, sticky (medium) */
/* Level 3: Floating ‚Äî modal, sheet, popover (light, transparent) */

--glass-blur-1: 24px;
--glass-blur-2: 16px;
--glass-blur-3: 12px;

--glass-saturate-1: 160%;
--glass-saturate-2: 180%;
--glass-saturate-3: 200%;

--glass-bg-1: color-mix(in srgb, var(--surface) 85%, transparent);
--glass-bg-2: color-mix(in srgb, var(--surface) 75%, transparent);
--glass-bg-3: color-mix(in srgb, var(--surface) 65%, transparent);

--glass-border-1: var(--border);
--glass-border-2: color-mix(in srgb, var(--border) 60%, transparent);
--glass-border-3: color-mix(in srgb, var(--border) 40%, transparent);
```

Then replace the old classes:

```css
.glass-structural {
  backdrop-filter: blur(var(--glass-blur-1)) saturate(var(--glass-saturate-1));
  -webkit-backdrop-filter: blur(var(--glass-blur-1)) saturate(var(--glass-saturate-1));
  background: var(--glass-bg-1);
  border-color: var(--glass-border-1);
}

.glass-elevated {
  backdrop-filter: blur(var(--glass-blur-2)) saturate(var(--glass-saturate-2));
  -webkit-backdrop-filter: blur(var(--glass-blur-2)) saturate(var(--glass-saturate-2));
  background: var(--glass-bg-2);
  border-color: var(--glass-border-2);
}

.glass-floating {
  backdrop-filter: blur(var(--glass-blur-3)) saturate(var(--glass-saturate-3));
  -webkit-backdrop-filter: blur(var(--glass-blur-3)) saturate(var(--glass-saturate-3));
  background: var(--glass-bg-3);
  border-color: var(--glass-border-3);
}
```

**Files:** `src/index.css`

### Step 4: Fix Card variant="glass" ‚Äî make it honest

Card variant="glass" currently has no blur. **Decision: Option A ‚Äî rename + add real glass.**

Rename current `"glass"` ‚Üí `"surface"` (honest name: bg-surface-1 + border + shadow). Add new `"glass"` variant that applies `glass-elevated` blur.

```tsx
// New glass variant ‚Äî actually frosts
glass: {
  className: `${BASE} border border-border-custom/45 glass-elevated`,
  style: { borderRadius: 'var(--radius-lg)', boxShadow: 'var(--shadow-card)' },
},
// Old "glass" renamed to "surface" (no blur, just surface + border + shadow)
surface: {
  className: `${BASE} border border-border-custom/45 bg-surface-1`,
  style: { borderRadius: 'var(--radius-lg)', boxShadow: 'var(--shadow-card)' },
},
```

Migration for existing consumers:
- All current `variant="glass"` consumers ‚Üí `variant="surface"` (no behavior change)
- MuscleHeatmap + BrainHealth (which manually added backdrop-blur) ‚Üí `variant="glass"` + drop manual blur

**Files:** `src/components/ui/Card.tsx`, `docs/DESIGN_SYSTEM.md`, all Card variant="glass" consumers

## What this plan does NOT do (intentionally)

- **Legacy variable migration** ‚Äî 654 variables across 1,134 references. This is the real 7‚Üí10 work but too large for one PR. Each `--legacy-arbitrary-*` needs semantic replacement (e.g., `--legacy-arbitrary-002` = `0.2em` tracking ‚Üí `--tracking-tight`). Should be a separate, incremental effort.
- **Z-layer restructure** ‚Äî current 12-token system is clean and well-used. No changes needed.
- **Blur token restructure** ‚Äî current 9-token system is clean. Only removing the dead `--blur-chrome`.
- **Component API changes** ‚Äî Button, Badge, Tabs, Input APIs are solid. No changes.
