## Decyzje projektowe

1. **Projects dostaje text search** — pattern jest dostępny przez `WorkspaceHeader.search`. Koszt dodania minimalny (wpiąć `searchQuery` w logikę filtrowania), a spójność z Todo/Keep/Links jest warta tę inwestycję.

2. **PillarFilterTabs → `WorkspaceHeader.secondaryRow`** — to jest content filter, nie navigation switcher ani view mode. `secondaryRow` jest semantic slot dla tego typu elementów (wzorzec: LinksInbox secondaryRow = URL input form).

3. **'Nowy cel' → `WorkspaceHeader.actions`** — primary variant, zgodnie z wzorcem Todo (push/history buttons w actions). 'Podsumowanie' → ghost variant.

4. **GrowthView i MedicalStudiesPage NIE wchodzą do WorkspaceNavigation** — to desktop-tier views z własnymi route'ami (`/rozwoj`, `/badania`) i własnym anchor-based subsection nav. Dodanie ich do WorkspaceNavigation pomieszałoby dwa paradigmy nawigacji.

5. **DesktopSectionNav active state → URL-hash based** — prostszy niż IntersectionObserver, wystarczy czytać `location.hash` + `location.pathname` (dla `/korelacje`). Hook `useActiveSection()` jest reużywalny.

### Commit 6: DesktopSectionNav — aktywny state

**Pliki:**
- `src/components/desktop/shell/DesktopSectionNav.tsx` (edytuj)
- Nowy: `src/components/desktop/shell/useActiveSection.ts`

**useActiveSection.ts:**
```ts
import { useLocation } from 'react-router-dom';

export function useActiveSection(sectionIds: string[]): string | null {
  const location = useLocation();
  // Check pathname first (for route-based sections like /korelacje)
  const pathMatch = sectionIds.find(id => location.pathname.includes(id));
  if (pathMatch) return pathMatch;
  // Then check hash
  const hash = location.hash.replace('#', '');
  if (hash && sectionIds.includes(hash)) return hash;
  // Default to first section
  return sectionIds[0] ?? null;
}
```

**DesktopSectionNav.tsx zmiany:**
1. Import `useActiveSection`
2. Zamień stały `linkClass` na funkcję `linkClass(id)` z warunkowym `nav-pill-active text-primary` dla aktywnego
3. Dodaj `aria-current="page"` na aktywnym linku
4. Dla `href` (Korelacje) — aktywny = `location.pathname === '/korelacje'`
